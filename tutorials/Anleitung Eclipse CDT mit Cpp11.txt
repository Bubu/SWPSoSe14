Tutorial für CDT Eclipse mit C++11:

Getestet mit
- Ubuntu 14.04 + GCC 4.8
- Win 8.1 + MinGW (neuste)

0. Schritt: neues Projekt erstellen(NICHT import verwenden)
	-use default location unchecken, Pfad selbst auswählen (Projekt-Compiler)
	- Ĺinux GCC als Toolchain(wenn nicht vorhanden: in Command-line: sudo apt-get install build-essential)

1. Schritt: -std=c++11 an 2 Stellen anhängen
	Project properties -> C/C++ Build -> Settings -> Tool Settings:
		-> GCC C++ Compiler --> Miscellaneous -> Other Flags --> -std=c++11 anhängen
		Zusätzlich -Wall und -Wextra -pedantic anhängen, da diese die Kompilierregeln verschärfen, worauf wir uns geeinigt haben.
		(Das Eingabefeld zu Other Flags gefüllt sah bei mir[Leon] dann so aus: -c -fmessage-length=0 -std=c++11 -Wall -Werror -pedantic )
		-> GCC C++ Linker --> Miscellaneous -> Other Flags --> -std=c++11 anhängen

2. Schritt: 2 Symbols hinzufügen
	Project properties -> C/C++ General -> Paths and Symbols -> Symbols -> GNU C++ -> Add...
	1. Symbol: 
		Name: __GXX_EXPERIMENTAL_CXX0X__
		Value: leer lassen
	2. Symbol:
		Name: __cplusplus
		Value: 201103L
	-> Apply

4. Schritt: 
	Window -> Preferences -> C/C++ -> Build -> Settings -> Tab:discovery:
	Auswählen: CDT GCC Build-in Compiler Settings [Shared]
	Unten in der Zeile "Command to get compiler specs:" vor "${INPUTS}" muss -std=c++11 eingefügt werden
	Meine Zeile sieht so aus: ${COMMAND} ${FLAGS} -E -P -v -dD  -std=c++11 "${INPUTS}"

5. Schritt:
	Project -> Clean
	Project -> C/C++ Index neu erzeugen
	evtl Eclipse neustarten (bei mir gings ohne)
